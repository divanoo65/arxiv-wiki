# 工具调用 (Tool Use)

### 概念定义
工具调用（Tool Use），在某些文献中也被称为函数调用（Function Calling），是指大型语言模型（LLM）在处理复杂任务时，能够识别出自身能力边界，并主动生成符合特定格式的指令，以调用外部工具或API来获取信息、执行计算或操作系统的能力。与传统的文本生成不同，工具调用要求模型输出结构化的数据（通常为JSON格式），这些数据被系统解析后，作为参数传递给预定义的函数。这一机制将模型从单纯的“文本预测器”转变为“智能代理（Agent）”，使其能够与真实世界的软件系统、数据库、文件系统及网络服务进行深度集成。在 Claude Code 等现代 AI 软件工程工具中，工具调用是实现自主编程、代码分析及环境交互的基石。

### 技术细节
工具调用的实现通常遵循以下技术流程：
1. **定义与注册**：开发者向模型提供一组工具的描述（Schema），包括函数名称、功能描述、参数类型及约束条件。
2. **意图识别与参数提取**：当用户输入指令时，模型分析上下文，判断是否需要调用工具。若需要，模型会生成一个包含函数名和参数的结构化请求。
3. **执行与反馈**：系统拦截该请求，在受控环境中执行对应的代码或API调用，并将执行结果（如成功返回的数据或错误信息）重新注入到模型的上下文窗口中。
4. **多轮循环**：模型根据工具返回的结果，决定是继续调用其他工具、进行下一步推理，还是直接向用户输出最终答案。

### 应用场景
- **自主软件开发**：如 [[Claude Code]] 中，模型通过调用文件读取、代码搜索、终端执行等工具，实现对代码库的自主修改与调试。
- **实时数据检索**：通过调用搜索引擎或内部数据库API，获取模型训练截止日期之后的信息。
- **复杂任务自动化**：在 [[AI 软件工程]] 流程中，自动执行测试用例、部署应用或进行日志分析。
- **环境交互**：通过调用操作系统接口，实现文件管理、进程控制等任务。
